## 0. 目标与原则

- **目标**:做出一个**有主页、能闯关、有输赢、能演示**的翻牌记忆游戏。
- **第一原则**:**完成 > 完美**。先保证全程可玩闭环,再谈打磨。任何功能卡住就降级或砍掉。
- **范围心法**:P0 决定"能不能交付",P1 决定"像不像样",P2 决定"有没有亮点"。**P0 没全绿之前,不碰 P1/P2。**

## 1. 技术栈(为速度选,不用 Phaser)

- **纯 HTML + CSS + 原生 JS,单文件 `index.html`**。
- **为什么**:翻牌游戏不需要游戏引擎;CSS Grid + CSS 3D 翻牌动画又快又顺,零构建、零依赖,浏览器双击即跑。
- **部署**:Vercel / Netlify 拖拽,或 GitHub Pages。

## 2. 页面与模式结构

```
Overview 主页
   └─ [闯关模式]  ← 当前唯一模式(结构预留可加更多模式)
        └─ 游戏页(逐关递进)
             └─ 结算页(过关 / 失败 / 全部通关)
```

- **Overview 主页**:标题 + 「开始闯关」按钮(+ 可选:最高关卡展示)。
- **模式可扩展**:当前只有"闯关模式";代码用"模式入口"结构组织,以后加无尽/限时模式只需多一个入口。
- **页面切换**:单页内 section 显隐(home / game / result),无需路由。

## 3. 核心玩法

**开局记忆阶段**:每关开始先**全部亮牌**供玩家记忆,锁输入,**此时间不计入 60s 游戏时间**;倒数结束后自动翻回,才开始计时与可操作。记忆时长**按网格大小递增:3×3→3s、4×4→4s、5×5→7s、6×6→10s**。
之后成对卡牌正面朝下,每次翻开两张:相同则配对保留,不同则翻回。
**在失败条件触发前**把当前关卡全部配对完成 → **过关,进入下一关**。失败 → 结算,可重试本关。

### 失败条件(任一触发即失败)
- **可错次数用尽(主难度杠杆)**:翻错累计达本关上限 → 失败。逐关收紧,直接考"记忆准不准"。
- **60 秒倒计时(固定兜底)**:每关限时 **60s**(常量,所有关一致),归零 → 失败。主要在大网格制造张力,小网格基本不触发。

### 难度模型(锯齿曲线,程序生成)
- 关卡按**网格尺寸**组织:**3×3 → 4×4 → 5×5 → 6×6**(逐级变大)。
- **每个尺寸的初始容错 = 网格边长 N**(3×3→3、4×4→4、5×5→5、6×6→6)。
- **同尺寸内,每过一个子关容错 −1,从 N 递减但最低到 3**(该尺寸共 N−2 个子关)。
- **升到更大尺寸时,容错重置到更高基数**——牌更多更难,故给更多容错,形成"紧→松一口气→再紧"的锯齿节奏。
- 关卡分布:3×3→1 关、4×4→2 关、5×5→3 关、6×6→4 关,**共 10 关**。
- **⚠️ 奇数网格留 1 个空位**(3×3 用 8 张=4 对,5×5 用 24 张=12 对),保证牌数为偶可配对,并保住正方形外观。
- 时间不参与难度调节(固定 60s),只做兜底/张力。

## 5. 关卡生成(替代手写关卡表)

```js
const MIN_N = 3, MAX_N = 6;     // 3×3 起步,6×6 结束
const MISTAKE_FLOOR = 3;       // 容错下限(降到 3 就不再降)
const TIME_LIMIT = 60;         // 每关固定限时(秒)
const PREVIEW_BY_N = {3:3, 4:4, 5:7, 6:10};  // 各网格开局记忆秒数(不计入 60s)

function buildLevels() {
  const levels = []
  let idx = 0
  for (let n = MIN_N; n <= MAX_N; n++) {
    const cells  = n * n
    const blanks = cells % 2            // 奇数网格留 1 空位
    const pairs  = (cells - blanks) / 2
    // 同尺寸:容错从 n 递减到 MISTAKE_FLOOR(共 n-2 个子关)
    for (let m = n; m >= MISTAKE_FLOOR; m--) {
      levels.push({ idx: idx++, n, cols: n, rows: n, blanks, pairs, maxMistakes: m, time: TIME_LIMIT })
    }
  }
  return levels
}
// 3×3: 容错[3] / 4×4: [4,3] / 5×5: [5,4,3] / 6×6: [6,5,4,3]  → 共 10 关
```
> 调难度/关卡量改 `MIN_N/MAX_N`;调时间松紧改 `TIME_LIMIT`,逻辑不动。

### 已实现的交互/视觉(index.html)
- **顶部大倒计时**:`clamp(44px,13vw,72px)` 居中,<10s 红色脉冲。
- **卡牌悬停**:整卡放大 1.06 + 蓝色光晕(`drop-shadow`);已配对卡不响应。
- **Target Cursor 瞄准镜光标**:四角框跟随鼠标、空闲缓慢旋转;**悬停卡片时四角弹开精准包裹该卡**、转正、变锁定红;离开棋盘隐藏、恢复系统光标。可调:`#cursor.lock{--cc}` 颜色、`CC.pad/W/off` 尺寸、`rot += 0.7` 转速。
- **格子大小**:`--board-max:600px`(调大=格子更大),手机自适应屏宽不溢出。
- **卡面白边**:`.cardimg{transform:scale(1.18)}` 裁掉素材自带白框。

### 卡片尺寸:固定,不随网格缩放(UX 决策)
- **所有关卡卡片尺寸恒定**,保证点击命中率与图案可读性在 3×3~6×6 完全一致。
- **尺寸按最大网格(6×6)能塞进屏幕宽度来定**:`卡片 = (min(视口宽, 棋盘最大宽) − 内边距 − 间距) / MAX_N`,所有网格复用此值。
- 效果:**6×6 铺满宽度;小网格(3×3)居中、四周留白;棋盘随网格变大,永不溢出屏幕**。
- 实现:CSS 变量 `--card` 固定,`grid-template-columns: repeat(cols, var(--card))`,棋盘容器居中。

### 卡面视觉:已接入 Cards/ 素材
- **已用 `Cards/sprite_1..20.png` 作为卡面**(20 张,够 6×6 的 18 对)。映射在 `FACE_POOL`:`Array.from({length:20},(_,i)=>'Cards/sprite_'+(i+1)+'.png')`。
- 渲染:正面用 `<img class="cardimg">`,`object-fit:cover` 填满方形卡片。
- 换素材只改 `FACE_POOL` 一处;配色/卡背/动画在 CSS `:root` 与 `.face` 中可调。
